Enrichment pipeline â€” from upload to inference

1. Upload & parse

Documents are uploaded via drag-and-drop or API. Guru parses PDFs, Markdown, JSON, YAML, and code files. Text is extracted, cleaned and normalized.

Format detection confidence

2. Chunk & embed

Text is split into semantic chunks (avg. 512 tokens). Each chunk is embedded via a 1536-dim model. Overlapping windows preserve context at boundaries.

3. Index & rank

Embeddings are indexed in a vector store with HNSW. At query time, top-k candidates are retrieved and re-ranked by cross-encoder for maximum precision.

Relevance scores â€” top-k retrieval
threshold 0.7

4. Inject & generate

Selected chunks are injected into the agent's context window with source metadata. The agent generates grounded responses with citations.
